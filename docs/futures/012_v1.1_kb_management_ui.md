# Knowledge Base Management UI

> **Versão Target:** V1.1
> **Status:** ⏳ Pendente
> **Owner:** Unassigned
> **Estimativa:** 25h

---

## Descrição

Atualmente requer execução manual de scripts para ingerir documentos. Não há visibilidade do que está no knowledge base. Esta feature cria UI para gerenciamento de KB.

---

## Passos de Implementação

### 1. Código

- [ ] `src/api/kb_routes.py`: CRUD endpoints para documentos KB
  - POST /api/kb/documents (upload)
  - GET /api/kb/documents (list)
  - DELETE /api/kb/documents/{id}
- [ ] `src/rag/knowledge_base.py`: Adicionar company_id metadata tagging
- [ ] `src/rag/ingestion.py`: Suportar upload via API
- [ ] Dashboard: Página de gerenciamento de KB
  - Upload de arquivos (PDF, DOCX, TXT, MD)
  - Lista de documentos com preview
  - Delete com confirmação

### 2. Testes

- [ ] Unit tests: KB CRUD operations
- [ ] Integration tests: Upload e retrieval E2E

### 3. Documentação

- [ ] `docs/KNOWLEDGE_BASE.md`: Guia de uso da UI

### 4. Infraestrutura

- [ ] File storage temporário (local ou S3)
- [ ] Background job para processamento de docs grandes
